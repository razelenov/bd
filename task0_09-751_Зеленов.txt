Зеленов Р.А. 751 (2)

Название темы: Электронный магазин.

1. Бизнес-требования:
Сайт электронного магазина обеспечивает возможности
	- посетителю сайта - подобрать вещь с подходящими характеристиками по лучшей цене;
	- посетителю сайта - сэкономить время поиска необходимого товара;
	- владельцу сайта - использовать дополнительный канал для продажи товаров.

2. Бизнес-правила:
Сайт электронного магазина должент соответствовать законам
	- Федеральный закон №152-ФЗ (ред. от 31.12.2017) "О персональных данных";
	- Закон РФ №2300-1 (ред. от 18.03.2019) "О защите прав потребителей".

3. Пользовательские требования:
Use Cases
	- Пошаговый сценарий для покупки выбранного товара.
User Stories
	- Добавление товара в корзину;
	- Онлайн-оплата выбранных товаров.

4. Атрибуты качества:
	- Время загрузки страницы не должно превышать 3 секунд при одновременном посещении сайта 30 пользователями;
	- Сайт должен быть адаптирован под основные мобильные операционные системы (Android, iOS, Windows Phone).

5. Ограничения:
	- Страницы сайта должны использовать язык разметки HTML версии 5.1 и выше;
	- Аутентификация пользователей должна происходить по вводу логина и пароля или номера телефона и пароля, тип введенных данных должен определяться автоматически.

6. Внешние интерфейсы:
	- Наличие API для массовой выгрузки имеющихся товаров и массовой загрузки новых позиций.
	- Поддержка прямых ссылок на изображения с различных ресурсов для публикации отзывов с фотографиями от клиентов.

7. Системные требования:
	- Доступ к API сервиса должен предоставляться с ряда ip-адресов, принадлежащих компании.

8. Функциональные требования:
	- На сайте должна быть реализована фильтрация товаров из выбранной категории по техническим характеристиками и стоимости продуктов.
	- На сайте должна быть реализована возможность сортировки товаров по цене с минимальной до максимальной и с максимальной до минимальной.
